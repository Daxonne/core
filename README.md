# Daxonne

> Language-agnostic code generation from your database schema.

Daxonne reads your database schema and generates production-ready code — DTOs, repositories, queries — through a template and plugin system. No ORM lock-in, no manual boilerplate.

[![CI](https://github.com/Daxonne/core/actions/workflows/ci.yml/badge.svg)](https://github.com/Daxonne/core/actions/workflows/ci.yml)
[![Go](https://img.shields.io/badge/Go-1.23+-00ADD8?logo=go)](https://go.dev)

## How it works

```
Database schema
      │
      ▼  daxonne pull
 schema.json (cached)
      │
      ▼  daxonne generate
 Handlebars templates
      │
      ▼
 Generated code (C#, TypeScript, Java, …)
```

1. **`daxonne pull`** — connects to your database, reads tables, columns, primary keys and foreign keys, and caches the schema locally.
2. **`daxonne add <template>`** — installs a code generation template.
3. **`daxonne generate`** — applies every template to every table and writes the output files.

```bash
go install github.com/daxonne/core@latest
```

```bash
git clone https://github.com/Daxonne/core.git
cd core
go build -o daxonne .
```

## Quick start

```bash
# 1. Initialise the project (creates daxonne.yaml)
daxonne init

# 2. Pull the schema from your database
daxonne pull

# 3. Install a code template
daxonne add csharp-dapper

# 4. Generate the code
daxonne generate
```

## Configuration — `daxonne.yaml`

```yaml
database:
  type: oracle                                          # oracle | postgres | mysql (future)
  connection: "oracle://user:password@host:1521/svc"
  owner: "MY_SCHEMA"

output:
  path: "./generated"

templates:
  - csharp-dapper
```

Copy `daxonne.yaml.example` as a starting point:

```bash
cp daxonne.yaml.example daxonne.yaml
# then edit with your connection string
```

> `daxonne.yaml` is gitignored — never commit credentials.

### Example output — `csharp-dapper`

**`CustomerDto.cs`**
```csharp
// Auto-generated by Daxonne — do not edit manually

public record CustomerDto(
    long Id,
    string FirstName,
    string LastName,
    string? Email,
    decimal Balance,
    DateOnly JoinedAt
);
```

**`CustomerRepository.cs`**
```csharp
// Auto-generated by Daxonne — do not edit manually

using System.Data;
using Dapper;

public class CustomerRepository
{
    private readonly IDbConnection _db;

    public CustomerRepository(IDbConnection db) => _db = db;

    public async Task<IEnumerable<CustomerDto>> GetAllAsync()
    {
        const string sql = @"SELECT ID, FIRST_NAME, LAST_NAME, EMAIL, BALANCE, JOINED_AT FROM CUSTOMER";
        return await _db.QueryAsync<CustomerDto>(sql);
    }

    public async Task<CustomerDto?> GetByIdAsync(long id)
    {
        const string sql = @"SELECT ID, FIRST_NAME, ... FROM CUSTOMER WHERE ID = :id";
        return await _db.QueryFirstOrDefaultAsync<CustomerDto>(sql, new { id });
    }

    // InsertAsync, DeleteAsync …
}
```

## Template format

Templates live in `.daxonne/templates/<name>/` and consist of:

```
template.json      ← manifest
dto.hbs            ← Handlebars template
repository.hbs     ← Handlebars template
…
```

**`template.json`**
```json
{
  "name": "my-template",
  "version": "1.0.0",
  "language": "csharp",
  "files": [
    { "template": "dto.hbs",  "output": "{{PascalCase name}}Dto.cs",  "per": "table" },
    { "template": "repo.hbs", "output": "{{PascalCase name}}Repo.cs", "per": "table" }
  ]
}
```

**`per`** can be `"table"` (one file per table) or `"schema"` (one file for the whole schema).

### Built-in Handlebars helpers

| Helper | Example input | Output |
|---|---|---|
| `{{PascalCase name}}` | `USER_ACCOUNT` | `UserAccount` |
| `{{CamelCase name}}` | `USER_ACCOUNT` | `userAccount` |
| `{{CSharpType type}}` | `decimal` | `decimal` |
| `{{JoinColumns columns}}` | — | `ID, FIRST_NAME, EMAIL` |
| `{{JoinParams columns}}` | — | `:Id, :FirstName, :Email` |
| `{{PrimaryKeyColumn columns}}` | — | `ID` |
| `{{PrimaryKeyType columns}}` | — | `long` |

### Template data shape (per table)

```json
{
  "name": "TABLE_NAME",
  "columns": [
    { "name": "ID", "type": "long", "nullable": false, "isPrimary": true },
    { "name": "EMAIL", "type": "string", "nullable": true, "isPrimary": false }
  ],
  "primaryKeys": ["ID"],
  "foreignKeys": [
    { "column": "DEPT_ID", "referencedTable": "DEPARTMENTS", "referencedColumn": "ID" }
  ]
}
```

### Internal type mapping

| InternalType | Oracle | C# |
|---|---|---|
| `string` | VARCHAR2, CHAR, CLOB | `string` |
| `int` | NUMBER(1–9) | `int` |
| `long` | NUMBER(10–18) | `long` |
| `decimal` | NUMBER(p,s>0), FLOAT | `decimal` |
| `date` | DATE | `DateOnly` |
| `datetime` | TIMESTAMP | `DateTime` |
| `bytes` | BLOB, RAW | `byte[]` |
| `uuid` | — | `Guid` |

## Architecture

```
core/
├── cmd/                    # Cobra CLI commands
├── internal/
│   ├── config/             # daxonne.yaml read/write
│   ├── schema/             # ISchemaReader interface + universal model
│   ├── generator/          # ICodeGenerator + Handlebars engine
│   ├── plugin/             # factory: DB type → ISchemaReader
│   └── template/           # registry + installer
├── plugins/
│   └── oracle/             # Oracle implementation (pure Go, no Oracle Client)
├── templates-src/          # source templates for daxonne add
└── .daxonne/
    └── templates/          # installed templates (local, gitignored cache)
```

Adding a new database is a matter of implementing `ISchemaReader`:

```go
type ISchemaReader interface {
    Connect(connString string) error
    ReadSchema(owner string) (*Schema, error)
    Close() error
}
```
